                                    Comandos Git

Para configurar el usuario y correo en el git es con los sgte comandos:
    git config --global user.name "NombreUsuario"
    git config --global user.email "nombreusuario@gmail.com"
    (si alguna vez hubo otro usuario registrado y ahora no te permite pushear ve al buscador de windows y pon)
        administrador de credenciales y elimina el usuario antiguo de tu github, y ahora configura tu nuevo usuario

Para confirmar que los valores se han establecido correctamente:
    git config --global --list

Inicializa un nuevo repositorio Git en el directorio actual:
    git init
 
Agrega el archivo texto.txt a la área de preparación (staging area):
    git add texto.txt
        sirve para agregar todos los archivos de uns vez
        git add .
    
Guarda (commitea) los cambios de la staging area en el historial local de Git:
    git commit -m "first commit"

Renombra la rama predeterminada de master a main
    git branch -M main

Establece una conexión entre tu repositorio local y el repositorio remoto en GitHub
    git remote add origin https://github.com/Juan/Repo1.git

     para ver los remotos configurados.
        git remote -v

Sube todos los commits locales de la rama main al repositorio remoto en GitHub.
    git push -u origin main

borrar commit sin borrar otros archivos
    git reset --soft HEAD-1

esto hace lo contrario de push, lo que hace pull es descargar cambios que esten en la nube, ejemplo un usuario agrego unas lienas mas al repositoria de github, lo que hace esto es sincronizar 
git pull origin master

Bifurcaciones (branches) con este comando vere todas las ramas que hay en mi repositorio
    git branch

crear una nueva rama
    git branch rama1

para entrar a la rama 
    git switch rama1                camando moderno
    git checkout rama2              comando antiguo

para fusionar ramas
    git merge

eliminar ramas 
    git branch -d caractersticaNueva

Corta tu rama rama1, la mueve y la "pega" justo encima del último commit que hay en master. Es como si acabaras de crear la rama hoy mismo, con todos los cambios nuevos ya incluidos
    git rebase rama1

Muestra las diferencias (cambios) entre dos puntos específicos del tiempo, generalmente entre un commit anterior y el estado actual o entre dos commits.
    git diff <hash>

Saca un archivo del área de preparación (staging) y lo devuelve al área de trabajo,  Se usa para "deshacer" un git add
    git restore --staged <archivo>

Muestra el historial de commits en una sola línea por entrada
    git log --oneline
